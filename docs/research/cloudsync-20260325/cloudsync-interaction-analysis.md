## 一、整体交互架构
Cloud Sync 的交互分成三层:

1. DSM 容器层
   - 先进入 DSM 登录页。
   - 登录成功后进入 DSM 桌面。
   - Cloud Sync 作为桌面应用窗口打开，不是独立站点页面跳转。

2. Cloud Sync 应用层
   - 顶部工具区: `新增`、`设置`
   - 左侧连接树: 当前观察到 `xinliu22 > WebDAV`
   - 右侧主工作区: `总览`、`任务列表`、`计划`、`设置`、`日志`

3. 二级弹窗/联动层
   - 顶部 `设置` 打开全局设置弹窗
   - 顶部 `新增` 打开接入向导
   - `总览 > 管理` 打开连接级操作菜单
   - `任务列表 > 打开` 会联动打开 DSM 的 File Station，并定位到本地目录

## 二、已观测页面与交互逻辑

### 1. 登录流程
路径:

1. 进入 `#/signin`
2. 输入用户名后进入 `#/signin/password`
3. 输入密码
4. 登录成功后回到 DSM 桌面

交互特点:
- 用户名和密码分两步输入。
- 登录完成后不是直接进入 Cloud Sync，而是进入 DSM 桌面，再由用户打开套件。

### 2. DSM 到 Cloud Sync 的进入路径
路径:

1. DSM 桌面展示应用图标网格
2. 点击 `Cloud Sync`
3. 以窗口形式打开 Cloud Sync

交互特点:
- Cloud Sync 与 DSM 其他应用并列，遵循 DSM 桌面式窗口模型。
- 应用窗口支持关闭、最小化、帮助等标准窗体动作。

### 3. Cloud Sync 主工作台
页面组成:
- 顶部工具按钮: `新增`、`设置`
- 左侧连接树: 账号/连接分层结构
- 右侧标签页: `总览`、`任务列表`、`计划`、`设置`、`日志`

交互特点:
- 先选左侧连接，再切右侧标签页查看连接详情。
- 所有标签页都围绕当前选中的连接展开。

### 4. 总览页
当前观察到的信息:
- 状态卡片显示 `最新的`
- 文案显示 `您的 Synology NAS 已是最新状态。`
- 显示连接信息: 公共云类型、用户名、服务器地址
- 有 `管理` 按钮

`管理` 按钮交互:
- 点击后展开下拉菜单
- 当前观察到两个动作:
  - `暂停同步`
  - `取消链接`

交互判断:
- `总览` 偏向连接健康状态和高风险操作入口。
- `管理` 是连接级动作集合，不直接在页面上暴露危险操作，先经由二级菜单确认入口。

### 5. 任务列表页
页面组成:
- 顶部操作条: `新增`、`编辑`、`打开`、`删除`
- 下方任务表格

当前观察到的任务:
- 本地路径: `/docker/ddns-go`
- 远程路径: `/我的备份`
- 状态: `最新的`
- 同步优先级: `中`

交互特点:
- 未选中行时，`编辑`、`打开`、`删除` 不可用。
- 选中任务行后，这些按钮变为可用。
- 点击 `打开` 后，会联动打开 File Station，并直接定位到任务对应的本地目录。

交互判断:
- 这是典型的“先选中，再操作”的桌面表格式交互。
- `打开` 并不是打开任务详情，而是跳到文件系统上下文，强调“同步对象的本地落点”。

### 6. 计划页
页面组成:
- `计划设置` 开关
- `暂停`、`运行` 两种状态按钮
- 7x24 小时矩阵
- `重置`、`应用` 按钮

当前观察:
- 计划开关未启用时，矩阵和控制区处于禁用态。
- 页面有提醒: `注意: 进入暂停模式会终止任务。`

交互判断:
- 计划页是连接级调度控制，不是任务级调度控制。
- 通过一周小时矩阵做可视化配置，适合桌面管理场景。

### 7. 连接设置页
页面组成:
- 常规设置
  - 连接名称
  - 轮询期(秒)
- 流量控制
  - 单个文件最大上传速率
  - 单个文件最大下载速率
- `重置`、`应用`

交互判断:
- 这里是当前连接的参数配置页。
- 粒度不是全局系统设置，而是连接级行为参数。

### 8. 日志页
页面组成:
- 顶部工具区: `打开`、搜索框、`高级搜索`
- 日志表格: 层级、日期与时间、名称、文件/文件夹、事件

当前观察到的日志样例:
- `2026.zip` / 文件 / 上传
- `.ddns_go_config.yaml` / 文件 / 上传

交互特点:
- 默认展示列表型操作日志。
- `打开` 按钮在未选中日志时不可用。
- 支持搜索和高级搜索，说明日志页承担问题追踪用途。

### 9. 顶部“设置”弹窗
定位:
- 这是 Cloud Sync 应用级全局设置，不是连接页中的“设置”标签

当前观察到的配置项:
- 管理员控制
  - 数据库位置设置
  - 同时传输的数量
  - 记录数上限
  - 管理员模式
- 常规
  - 套件重新激活/计划暂停后恢复时，对“本地已删除文件”的默认处理策略

交互判断:
- 这个弹窗管理的是 Cloud Sync 套件的全局运行策略。
- 页面使用弹窗而不是路由切换，说明它被定义为“辅助配置”，不是主工作区的一部分。

### 10. 顶部“新增”向导
已观测到的步骤:

1. 第一步: `云供应商`
   - 提供搜索框
   - 以双列表格展示云厂商/协议
   - 当前可见包括 WebDAV、Dropbox、Google Drive、OneDrive、S3 等

2. 第二步: `帐户设置`
   - 选中 `WebDAV` 后点击 `下一步`
   - 出现字段:
     - 服务器地址
     - 用户帐户
     - 密码

交互判断:
- 这是明显的分步向导模型: 先选协议/厂商，再填认证信息，后续应继续进入目录选择或任务配置。
- 当前只记录到第二步，未继续提交，避免改写现有系统配置。

## 三、核心交互规则总结

### 1. 对象层级
- 套件全局设置
- 连接
- 连接下的任务
- 任务映射到本地目录和远程目录

### 2. 行为触发方式
- 全局行为通过顶部按钮触发: `新增`、`设置`
- 连接行为通过 `总览 > 管理` 或标签页触发
- 任务行为通过 `任务列表` 中“先选中后操作”触发

### 3. 状态反馈方式
- 连接状态通过总览卡片展示，如 `最新的`
- 任务状态通过表格字段展示
- 历史动作通过日志表格沉淀

### 4. 产品交互风格
- 强桌面应用风格，不是移动端/单页 Web 站点风格
- 大量使用表格、树、工具栏、弹窗、二级菜单
- 风险动作隐藏在二级入口中
- 数据对象切换主要依赖左侧树与中部表格选择

## 四、推导出的典型使用路径

### 路径 A: 查看连接健康状态
1. 打开 Cloud Sync
2. 在左侧树选择连接
3. 默认进入 `总览`
4. 查看最新状态与连接信息
5. 如有需要，通过 `管理` 执行暂停或取消链接

### 路径 B: 管理同步任务
1. 进入 `任务列表`
2. 查看任务表格
3. 选中某一条任务
4. 执行 `编辑` / `打开` / `删除`
5. 若点击 `打开`，进入 File Station 查看本地文件

### 路径 C: 配置同步时间
1. 进入 `计划`
2. 开启 `计划设置`
3. 在 7x24 矩阵中设置运行时段
4. 点击 `应用`

### 路径 D: 新增云连接
1. 点击顶部 `新增`
2. 在 `云供应商` 中选择协议或厂商
3. 点击 `下一步`
4. 填写账户信息
5. 后续应继续完成授权/目录/任务配置
